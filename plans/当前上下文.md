# Momo 桌面端开发上下文

> 本文件只保留**项目方向、开发规范、验证命令、已知架构债**。
> 「当前实现到哪一步」请直接查 `git log` 与源码，不要把"阶段 N 已完成"之类的快照写回本文件。

## 项目方向

- Momo 桌面端定位为**前端优先工具**：UI、本地 SQLite、WebDAV 同步与本地任务 / 日程体验是主战场。
- 服务端能力（账号、Agent、排期、AI 调用）由独立项目承接，本仓库不再承担后端执行。
- 共享契约保留在 `packages/contracts/`，作为日后接入服务端的协议来源；`apps/api/` 已下线，不要恢复内存 router 作为默认设置页 runner。
- 旧路线中以 `BE-*` / `IF-*` 命名的后端 / 基础设施阶段不再适用；推进时按桌面端真实需求重新拟定阶段名。
- WebDAV 同步首选坚果云：应用密码、ETag 乐观锁、PROPFIND 逐层 list；其他 WebDAV 服务在坚果云稳定后再接入。

## 开发规范

- 每阶段修改前先确认 `git status --short --branch`。
- 测试驱动：先写失败测试，再实现；测试文件优先放 `apps/desktop/tests/`。
- 业务数据只走 `TaskRepository`，不要在页面直接写 SQL。
- 保留 Rust `greet` smoke test；除 Tauri 插件 / 窗口能力外，不把业务逻辑搬到 Rust。
- 不重写历史，不回滚用户或已提交改动。
- 提交信息使用中文；PR 描述、文档、对话同样使用中文；代码标识符仍用英文。
- 不要新增自动冲突解决、被拒绝变更自动重试、本地变更删除或强制 mark synced 等动作；同步诊断卡片保持只读。
- 不要在没有用户授权的情况下启动后台或定时 WebDAV 同步。

## 验证命令

在 `apps/desktop/`：

```bash
npm run test
npm run build
```

在 `apps/desktop/src-tauri/`：

```bash
cargo check
```

触碰 `packages/contracts/` 时，在仓库根目录追加：

```bash
.\apps\desktop\node_modules\.bin\tsc.cmd -p packages\contracts\tsconfig.json
```

仓库根的 `npm run verify` 会串行跑桌面端 test / build、Tauri cargo check、contracts tsc。提交前必须确认相关命令 exit 0。

## 已知架构债（推进时优先考虑）

- `WebdavSettingsCard.vue` 拆 `WebdavCredentialsForm` + `WebdavSyncStatus` + `useWebdavSync()` composable；Settings.vue 用 controller.onRunCompleted 订阅替代 `@sync-done` 事件。
- `snapshot.ts` 已实现合并，但 `provider.snapshot()` 只 `listEntities` 没启用 snapshot 文件；引入 calendar 之前需把 snapshot 触发条件（oplog > 1k）与 cursor 复位接到 scheduler。
- 自动调度 / 后台同步触发层（cron-like）尚未实现。
- 多 provider WebDAV（Nextcloud / OwnCloud / 自建 dav）：抽离 `secretsToConfig` 与 `paths.ts` 的特化点。
- `taskBridge` 升级为通用 `entityBridges` 注册表（`EntityBridge<TPayload>` 接口：`entityType` / `schemaVersion` / `localChangeToOp` / `entityToLocal`），为后续 calendar / agent-task 等实体留扩展位。
