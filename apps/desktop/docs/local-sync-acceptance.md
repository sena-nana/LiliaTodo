# WebDAV 同步验收清单

## 范围

本清单验证桌面端设置页的 WebDAV 同步主路径。它使用真实 WebDAV 服务、本地 SQLite repository 和 `@tauri-apps/plugin-http` 的网络出口；目标是确认凭据持久化、立即同步、用户授权后的自动后台同步、报告卡片与设置页顶层卡片刷新形成闭环，并能在两台设备之间真正同步任务与清单。

## 前置准备

- 拥有可访问的 WebDAV 账号（首选坚果云，使用应用密码而非主密码）。
- 已运行 `npm install`，可启动 `npm run tauri dev`。
- 准备两台机器或同一机器上的两份独立 SQLite 数据库副本，用于验证多端同步。

## Vite 冒烟（仅可访问性）

- 运行 `npm run dev`。
- 打开 `http://localhost:1420/settings`。
- 确认 `WebDAV 同步（坚果云优先）` 卡片可见，表单字段（Base URL / 根目录 / 用户名 / 应用密码 / 设备 ID）均可键盘聚焦。
- 确认页面已不再显示 `本地同步模拟` 按钮或 `远程同步配置（旧 HTTP 通路）` 卡片。
- 注意：普通浏览器没有 `tauri-plugin-store` 与 `tauri-plugin-http`，凭据保存与立即同步会直接报错，Vite 冒烟不验证完整链路。

## Tauri WebView 完整 SQLite + WebDAV 流程

- 运行 `npm run tauri dev`。
- 进入设置页，填写 WebDAV 凭据（Base URL、用户名、应用密码、设备 ID）并保存。
- 确认提示 `已保存到本机安全存储`；再次打开设置页应自动回填除密码外的字段。
- 创建或编辑若干任务，再新增一个清单并在该清单内放入至少一个任务，让 `待同步` 大于 0。
- 在 WebDAV 卡片点击 `立即同步`。
- 同步完成后确认：
  - WebDAV 卡片在按钮下方追加一行人类可读的同步结果 message（例如 `已上传 N 条本地任务变更，已上传 M 个本地清单变更，已应用 K 个远端清单`）。
  - 主界面（今日 / 收件箱 / 日历）能立即看到对端新增的任务。
  - 侧边栏能立即看到对端新增的清单。
- 在第二台机器（或同一机器换 SQLite 数据库副本）配置同一组 WebDAV 凭据并点击 `立即同步`，确认能拉到第一台机器创建的任务和清单。
- 在第一台机器归档刚创建的清单并再次点击 `立即同步`；第二台机器再次同步后，确认该清单不再出现在侧边栏，原清单内任务已迁移回收件箱。
- 在设置页开启 `自动后台同步`，确认卡片显示已启用并立即跑一次同步；重启桌面端后应恢复已授权状态并再次启动同步。
- 自动后台同步开启后创建或编辑任务，等待 idle 防抖窗口后确认会触发同步；关闭 `自动后台同步` 或清除 WebDAV 凭据后，本地变更不应再触发后台同步。

## 日历与 Agent 验收边界

- 打开日历页，确认可在 `日` / `周` / `月` 视图之间切换，任务可打开详情；拖拽到其他日期后任务截止时间随之更新。
- 日历页有容量负载、冲突检测和排期建议；不要再按旧的 7 天只读口径验收。
- 打开 `Agent 收件箱`，确认页面包含 runtime 状态、待确认操作、工具目录、执行批次与运行事件。
- 手动触发扫描后，Agent 建议只进入待确认队列；用户确认后才执行并写入审计记录，拒绝或撤销不会反向触发自动扫描。
- 在设置页关闭 Agent 自动触发后，任务写入、提醒到期和每日启动不应生成自动 envelope；runtime 未运行或未配置 backend 时只记录诊断，不应写入待确认操作。

## 失败路径

- 故意填错应用密码并保存；点击 `立即同步` 后确认 WebDAV 卡片显示 `错误：WebDAV ...` 文案。
- 把网络断开（关闭 Wi-Fi 或停掉网络代理），点击 `立即同步` 后确认 WebDAV 卡片显示同类错误文案；底层 `sync_state.lastError` 会被 runner 写回，便于通过日志或调试手段排查。

## 回归护栏

- 不要新增自动冲突解决、被拒绝变更重试、本地变更删除或强制标记已同步操作。
- 不要在默认设置页恢复 `本地同步模拟` 或旧 `远程同步配置（旧 HTTP 通路）` 卡片。
- `apps/api/` 已整体下线，不要恢复其内存 router 作为设置页默认 runner；如需真接入后端，请基于 `packages/contracts` 的 DTO 在独立服务端项目实现。
- WebDAV 后台或定时同步必须由用户在设置页显式授权，关闭开关或清除凭据后不得继续后台触发。
- 不要把 `服务端游标`、`同步历史`、`待同步变更原始 payload` 等开发诊断信息回填到设置页面 UI，这些属于调试范畴。
